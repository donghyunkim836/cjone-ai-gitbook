# ❺ 대화 맥락 관리하기

Claude는 한 번에 처리할 수 있는 대화량에 한계가 있습니다. 이걸 "컨텍스트 윈도우"라고 합니다. \
회의가 길어지면 처음에 했던 얘기가 흐려지는 것처럼, 대화가 길어질수록 앞부분의 정확도가 떨어집니다.

크기가 정해진 메모장이라고 생각하면 됩니다. \
꽉 차면 새 메모장으로 시작하거나, 핵심만 추려서 공간을 확보해야 합니다.

`/clear`와 `/compact`, 두 명령어만 알면 대부분 해결됩니다.

## ⏵ /clear vs /compact

|    | `/clear`        | `/compact`      |
| -- | --------------- | --------------- |
| 효과 | 대화 기록 완전 초기화    | 핵심만 남기고 압축      |
| 언제 | 완전히 새 작업을 시작할 때 | 같은 작업을 계속 이어갈 때 |
| 비유 | 메모장을 새로 펼치기     | 메모장 여백 확보하기     |

### /clear

다른 작업으로 넘어갈 때 씁니다. 이전 맥락이 남아 있으면 Claude가 엉뚱한 방향으로 갈 수 있습니다.

```
이 프로젝트의 README.md를 한국어로 번역해줘
```

번역이 끝나면 `/clear`로 맥락을 정리하고 새 작업을 시작합니다.

```
/clear

이 보고서의 3월 매출 데이터를 분석해줘
```

`/clear` 이후에는 이전 대화를 모릅니다. "아까 그 파일"이라고 하면 뭔지 알 수 없으니 구체적으로 다시 말해줘야 합니다.

참고로 `/clear`는 대화 기록만 지우는 거고 세션 자체를 끝내는 건 아닙니다. 세션을 완전히 새로 시작하려면 `/exit` 후 `claude`를 다시 실행합니다.

Desktop 앱에서 새 맥락으로 시작하려면 새로 만들기 버튼을 클릭합니다.

* Chat → New chat
* Cowork → New task
* Code → New session

### /compact

같은 작업을 계속하면서 맥락이 너무 쌓였을 때 씁니다.

```
/compact
```

유지할 내용을 지정하면 더 정확하게 압축됩니다.

```
/compact API 변경사항 위주로 정리해줘

/compact 수정한 파일 목록과 남은 작업을 남겨줘
```

압축 후 남는 것과 사라지는 것이 있습니다.

| 다시 읽어 재주입        | 요약에 포함        | 사라짐                     |
| ---------------- | ------------- | ----------------------- |
| 시스템 프롬프트         | 요청 내용과 의도     | 전체 도구 출력 (파일 내용, 명령 결과) |
| CLAUDE.md        | 수정한 파일과 주요 코드 | 중간 추론 과정                |
| Auto Memory      | 에러와 해결 방법     | 스킬 목록 (사용한 것만 보존)       |
| 환경 정보, MCP 도구 목록 | 남은 작업과 현재 상태  |                         |

> 대화 중에만 한 지시는 압축 때 사라질 수 있습니다. "한국어로 응답해줘" 같은 규칙은 CLAUDE.md에 적어두세요.

## ⏵ 맥락 용량 확인 (/context)

```
/context
```

전체 용량 대비 현재 사용량을 확인할 수 있습니다. **70%를 넘으면 `/compact`를 고려**합니다.

대화 내용 외에도 시작부터 컨텍스트를 차지하는 항목들이 있습니다.

| 항목            | 내용                     | 로드 시점 |
| ------------- | ---------------------- | ----- |
| System prompt | Claude 기본 동작 지시        | 세션 시작 |
| System tools  | 파일 읽기, 편집 등 도구 정의      | 세션 시작 |
| Memory files  | CLAUDE.md, Auto Memory | 세션 시작 |
| Skills        | 스킬 목록 (전체 내용은 사용 시 로드) | 세션 시작 |
| MCP servers   | 연결된 서버의 도구 정의          | 세션 시작 |
| Messages      | 대화 내용, 파일 내용, 명령 결과    | 대화 중  |

CLAUDE.md가 길거나 MCP 서버가 많으면 기본 사용량이 올라갑니다. `/mcp`로 서버별 토큰 사용량을 확인하고 안 쓰는 서버는 연결을 끊는 게 좋습니다.

## ⏵ 자동 압축

맥락이 95% 차면 자동으로 압축이 실행됩니다. 95%까지 쌓인 다음 압축하는 것보다, 50% 수준에서 미리 `/compact`를 쓰는 게 비용이 덜 들고 품질도 좋습니다. 직접 압축하면 유지할 내용도 지정할 수 있으니까요.

같은 문제를 두 번 이상 수정했다면 실패한 시도들이 맥락을 어지럽히고 있는 겁니다. `/clear` 후 새 프롬프트로 다시 시작하는 게 더 효과적일 수 있습니다.

## ⏵ 자동 압축 시점 조절

더 일찍 압축하고 싶으면 환경변수로 기준을 낮출 수 있습니다.

```bash
# 50%에서 자동 압축 실행 (기본값: 95%)
export CLAUDE_AUTOCOMPACT_PCT_OVERRIDE=50

# 1M 모델을 500K 기준으로 운용
export CLAUDE_CODE_AUTO_COMPACT_WINDOW=500000
```

## ⏵ 압축 때 유지할 내용 지정

CLAUDE.md에 압축 지시를 적어두면 자동 압축에서도 반영됩니다.

```markdown
# Compact Instructions

압축 시 수정된 파일 목록과 테스트 명령어를 항상 남길 것
```

`/compact`나 `/clear` 후에도 CLAUDE.md는 디스크에서 다시 읽어 주입됩니다. 매번 반복하게 되는 규칙은 대화가 아닌 CLAUDE.md에 적어두세요.
