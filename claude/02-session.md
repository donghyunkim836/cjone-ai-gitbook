# 02. 대화와 세션

설치를 마쳤으면 매일 쓰게 될 기본 조작법을 익혀봅니다.

## 대화 시작하기

CLI에서 `claude`를 실행하면 새 세션이 시작됩니다. 첫 메시지를 바로 붙여서 실행할 수도 있습니다.

```bash
# 대화형 세션 시작
claude

# 첫 메시지와 함께 시작
claude "이 프로젝트 구조를 설명해줘"
```

Desktop 앱은 Code 탭에서 프로젝트 폴더를 선택한 뒤 바로 입력하면 됩니다.

## 세션 이어하기

세션을 종료한 뒤에도 이전 대화를 이어갈 수 있습니다.

### 1. 마지막 대화 이어하기

`/exit`으로 종료한 뒤 `-c` 플래그로 다시 시작하면 직전 대화가 그대로 이어집니다.

```bash
claude -c
```

### 2. 세션에 이름 붙이기

대화 중에 `/rename`으로 이름을 붙여두면 나중에 찾기 편합니다.

```
/rename 버그수정
```

### 3. 이름으로 다시 시작하기

```bash
claude -r "버그수정"
```

이전 세션 목록 전체를 보려면 대화 중에 `/resume`을 입력합니다.

## 파일 참조 (@)

대화 중에 특정 파일을 언급하고 싶을 때 `@`를 입력합니다.

```
@src/main.js 이 파일의 구조를 설명해줘
```

`@`를 입력하면 자동완성으로 파일 목록이 뜹니다. 경로를 외울 필요 없이 선택하면 됩니다.

## 슬래시 명령어

`/`를 입력하면 사용 가능한 명령어 목록이 나타납니다. 자주 쓰는 것만 정리했습니다.

| 명령어 | 역할 |
|--------|------|
| `/help` | 도움말 |
| `/exit` | 세션 종료 |
| `/model` | 모델 변경 (Opus, Sonnet, Haiku) |
| `/voice` | 음성 입력 모드 (스페이스바 길게 눌러 녹음) |
| `/resume` | 이전 세션 목록에서 선택해서 재개 |
| `/rename` | 현재 세션에 이름 붙이기 |
| `/clear` | 대화 기록 초기화 |
| `/compact` | 대화를 핵심만 남기고 압축 |

지금 다 외울 필요는 없습니다. 쓰다 보면 자연스럽게 익숙해집니다. `/`를 입력하면 언제든 전체 목록을 확인할 수 있습니다.

## 여러 줄 입력

`Enter`를 누르면 바로 전송되기 때문에 줄바꿈할 때는 별도 단축키를 씁니다.

| 방법 | 단축키 |
|------|--------|
| 모든 터미널 | `\` + Enter |
| macOS 기본 | Option+Enter |
| iTerm2, WezTerm, Ghostty, Kitty | Shift+Enter |

그 외 터미널에서 Shift+Enter를 쓰려면 `/terminal-setup`을 먼저 실행합니다.

## 되돌리기 (체크포인트)

Claude가 파일을 수정할 때마다 자동으로 체크포인트가 만들어집니다. 게임 세이브 포인트와 같습니다. 결과가 마음에 들지 않으면 `Esc`를 두 번 누릅니다.

```
Esc Esc
```

되감기 메뉴가 나타나면 화살표 키로 돌아갈 시점을 선택합니다.

- **Restore code and conversation** - 코드와 대화 모두 해당 시점으로 되돌림
- **Restore code** - 파일만 되돌리고 대화 맥락은 유지 (가장 자주 씀)
- **Restore conversation** - 대화만 되돌리고 파일은 그대로
- **Summarize from here** - 해당 시점부터 대화를 요약으로 압축
- **Never mind** - 취소

`/rewind`를 입력해도 같은 메뉴가 나타납니다. 횟수 제한 없이 자유롭게 되돌릴 수 있습니다.

## 세션 보관 기간 설정

기본값은 30일 후 자동 삭제입니다. 세션 기록을 오래 보관하고 싶다면 Claude에게 바로 부탁하면 됩니다.

```
세션 보관 기간을 365000일로 설정해줘. ~/.claude/settings.json의 cleanupPeriodDays 값을 변경해줘.
```

직접 수정하려면 `~/.claude/settings.json`을 열고 아래 값을 추가합니다.

```json
{
  "cleanupPeriodDays": 365000
}
```

`0`으로 설정하면 시작할 때마다 기록이 초기화됩니다.
