# ❻ 기존 규칙 유지하기

Claude Code는 세션을 종료하면 백지 상태가 됩니다. 이전에 나눈 대화, 정해둔 규칙을 전혀 기억하지 못합니다.

한 시간 동안 프로젝트 구조를 설명하고 코딩 스타일을 맞췄는데, \
다음 세션에서 다시 처음부터. 이런 경험이 반복되면 AI를 쓰는 이유가 없어집니다.

CLAUDE.md가 이 문제를 해결합니다. \
프로젝트 폴더에 두는 마크다운 파일로, 세션이 시작될 때 자동으로 읽어 맥락에 넣어줍니다. \
작업 규칙이나 프로젝트 정보를 한 번만 적어두면 매번 다시 말할 필요가 없습니다.

## ⏵ /init으로 자동 생성

프로젝트 폴더에서 Claude Code를 실행하고 `/init`을 입력합니다.

```
/init
```

Claude가 파일 구조, package.json, README 등을 분석해서 CLAUDE.md 초안을 만들어줍니다. 이미 파일이 있으면 덮어쓰지 않고 개선안을 제안합니다.

## ⏵ 직접 작성하기

자동 생성보다 직접 쓰는 게 더 정확할 때가 많습니다.

**비즈니스 업무용 예시:**

```markdown
# 마케팅팀 업무 자동화

## 작업 규칙
- 응답은 한국어로 작성
- 보고서는 격식체(합니다/습니다) 사용
- 금액 표기: 원화 기호 없이 숫자만, 천 단위 쉼표 포함 (예: 1,200,000)
- 날짜 형식: YYYY-MM-DD

## 자주 쓰는 작업
- 주간 보고서: 성과 요약 + 다음 주 계획 + 이슈 사항
- 이메일 분류: 긴급도(높음/중간/낮음) + 주제별 분류

## 참고 정보
- 팀장: 박팀장 (보고 대상)
- 주간 회의: 매주 월요일 오전 10시
- 보고서 마감: 매주 금요일 오후 5시
```

**개발 프로젝트용 예시:**

```markdown
# 프로젝트 개요
할일 관리 REST API - 사용자별 CRUD 및 카테고리 분류

## 기술 스택
- Node.js 20 + Express
- PostgreSQL + Prisma
- Jest + Supertest

## 명령어
- 개발 서버: `npm run dev`
- 전체 테스트: `npm test`
- 린트: `npm run lint`

## 코드 규칙
- require 대신 import/export 사용 (ES Modules)
- 에러 핸들링은 express-async-errors 사용
- .env 파일은 커밋하지 않음
```

## ⏵ 잘 쓰는 법

Claude Code 창시자 Boris Cherny가 팀에서 쓰는 방식입니다.

> "Claude가 실수할 때마다 CLAUDE.md에 추가한다. 그러면 다음에는 같은 실수를 하지 않는다."

**넣을 것**

* Claude가 스스로 알 수 없는 명령어 (빌드, 테스트, 린트)
* 기본값과 다른 코드 스타일 규칙
* 브랜치 네이밍, PR 규칙 같은 팀 관례
* 프로젝트 고유 아키텍처 결정
* 문서 형식, 말투, 반복되는 업무 패턴

**빼야 할 것**

* 코드를 읽으면 알 수 있는 것
* 언어 표준 관례 (Claude가 이미 알고 있음)
* 자주 바뀌는 정보
* "깔끔하게 작성"처럼 모호한 지시

한 줄을 쓸 때마다 "이걸 빼면 Claude가 실수할까?"를 따져보세요. 아니라면 빼는 게 낫습니다. 파일 하나당 200줄 이내가 적당합니다. 길수록 중요한 규칙을 흘려읽게 됩니다.

200줄이 넘으면 별도 파일로 분리하고 `@` 구문으로 가져옵니다.

```markdown
# Git 워크플로우
@docs/git-instructions.md

# 개인 설정 (Git에 포함되지 않음)
@~/.claude/my-project-instructions.md
```

API 키나 비밀번호는 절대 넣지 마세요. CLAUDE.md는 Git에 커밋되기 때문에 외부에 노출됩니다.

## ⏵ 범위 이해

CLAUDE.md는 세 곳에 둘 수 있고, 세션 시작 시 세 곳을 모두 읽어 합칩니다.

| 범위    | 위치                    | 용도      | Git   |
| ----- | --------------------- | ------- | ----- |
| 프로젝트  | `./CLAUDE.md`         | 팀 공통 규칙 | 포함    |
| 개인    | `~/.claude/CLAUDE.md` | 개인 선호   | 미포함   |
| 조직 정책 | 시스템 경로                | 전사 정책   | 별도 관리 |

개인 CLAUDE.md는 내 컴퓨터에만 적용되어 모든 프로젝트에 공통으로 반영됩니다.

```
~/.claude/CLAUDE.md 파일을 만들어줘. 내용은 이렇게:
- 응답은 한국어로
- 커밋 메시지는 한국어로 작성
- 코드 변경 시 관련 테스트도 함께 수정
- 변수명과 함수명은 영어 사용
```

## ⏵ Auto Memory

CLAUDE.md는 내가 직접 쓰는 지시서입니다. Auto Memory는 Claude가 작업하면서 스스로 메모하는 기능입니다.

|     | CLAUDE.md    | Auto Memory     |
| --- | ------------ | --------------- |
| 작성자 | 사람           | Claude          |
| 내용  | 지시와 규칙       | 작업 중 파악한 패턴     |
| 범위  | 프로젝트, 개인, 조직 | 프로젝트별 (내 컴퓨터에만) |
| 용도  | 코드 표준, 워크플로우 | 빌드 명령어, 디버깅 패턴  |

예를 들어 Claude가 반말로 보고서를 썼을 때 "격식체로 써줘"라고 고쳐주면, Claude가 스스로 메모해 다음 세션부터는 알아서 격식체로 씁니다.

Auto Memory는 기본으로 켜져 있으며, 유용하다고 판단될 때만 저장합니다. `/memory`를 입력하면 저장된 내용을 확인하거나 수정할 수 있습니다.

## ⏵ 동작 확인

설정 후 Claude를 새로 실행하고 확인해봅니다.

```
이 프로젝트의 작업 규칙이 뭐야?
```

CLAUDE.md에 적힌 내용을 인용하며 답하면 정상입니다.
