# 03. 모델과 Effort

Claude Code에서는 모델과 사고 깊이(effort)를 선택할 수 있습니다. 가능하면 가장 좋은 모델, 가장 높은 effort 레벨을 쓰면 결과가 좋습니다. 하지만 높을수록 토큰을 많이 쓰고, 토큰은 곧 비용입니다. 업무에 따라 조절해서 사용하는 것을 추천합니다.

## 모델 선택

`/model`을 입력하면 사용 가능한 모델 목록이 나타납니다.

| 모델 | 특징 |
|------|------|
| Opus 4.6 (1M context) | 가장 강력. 복잡한 작업에 적합 (기본값, 권장) |
| Sonnet 4.6 | 일상적인 작업에 균형 잡힌 선택 |
| Sonnet 4.6 (1M context) | Sonnet에 확장 컨텍스트. 추가 비용 발생 ($3/$15 per Mtok) |
| Haiku 4.5 | 가장 빠름. 간단한 질문에 적합 |

화살표 키로 선택하고 Enter로 확인합니다. 선택한 모델은 이번 세션과 이후 세션에도 적용됩니다. CLI에서는 `--model` 플래그로도 지정할 수 있습니다.

```bash
claude --model sonnet
```

`/fast`를 입력하면 Fast mode를 켤 수 있습니다. Opus 모델이 더 빠른 출력 속도로 동작하며, 추가 비용이 발생합니다. 즉, 토큰이 빠르게 소진됩니다. v2.1.142부터 Fast mode 기본 모델이 Opus 4.7이며, Opus 4.6에 고정하려면 `CLAUDE_CODE_OPUS_4_6_FAST_MODE_OVERRIDE=1`를 설정합니다.

## Effort 레벨

같은 모델이라도 사고 깊이를 조절할 수 있습니다.

| 레벨 | 적합한 작업 | 사고 깊이 |
|------|------------|-----------|
| low | 간단한 질문, 오타 수정, 파일 이름 변경 | 얕고 빠름 |
| medium | 일반적인 코드 작성, 문서 정리 | 균형 |
| high | 복잡한 버그 분석, 아키텍처 설계, 여러 파일 리팩토링 (Opus 4.6/Sonnet 4.6 기본값) | 깊고 느림 |
| xhigh | 요구 난이도 높은 코딩/에이전트 작업 (Opus 4.7 전용, 기본값) | 매우 깊음 |
| max | 가장 깊은 사고, 토큰 제한 없음 (현재 세션만 적용) | 최대 |

설정 방법은 두 가지입니다.

```
/effort [low|medium|high|xhigh|max|auto]
```

또는 `/model`을 열면 모델 선택과 함께 하단에 effort 슬라이더가 나타납니다. `← →` 키로 조절하고 Enter로 확인합니다.

`/effort auto`로 모델 기본값으로 되돌릴 수 있습니다. 현재 effort 레벨은 스피너 옆에 "with low effort"처럼 표시됩니다.

## ultrathink

한 번만 깊이 생각하게 하고 싶을 때는 메시지에 `ultrathink`를 포함하면 그 턴에만 high 수준이 적용됩니다. 문장 앞, 뒤, 중간 어디에 있어도 됩니다.

```
ultrathink 이 인증 로직에서 race condition이 발생할 수 있는 경로를 모두 분석해줘
```
