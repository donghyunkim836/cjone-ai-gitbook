# 05. 컨텍스트 관리

Claude는 한 번에 처리할 수 있는 대화량에 한계가 있습니다. 이것을 "컨텍스트 윈도우"라고 부릅니다. 회의가 3시간째 이어지면 처음에 한 얘기가 흐릿해지는 것처럼, Claude도 대화가 길어지면 앞부분의 정확도가 떨어집니다.

사이즈가 정해진 작업 노트라고 생각하면 됩니다. 모델마다 노트 사이즈가 다릅니다. 1만자짜리 노트부터 100만자짜리 노트까지 다양하지만, 어떤 노트든 다 쓰면 새 노트로 시작하거나, 오래된 내용을 지우고 핵심만 요약해서 다시 적어야 합니다.

`/clear`(새 노트로 시작)와 `/compact`(요약해서 공간 확보) 두 명령어만 알면 됩니다.

## /clear와 /compact

| | `/clear` | `/compact` |
|--|---------|-----------|
| 효과 | 대화 이력 완전 초기화 | 핵심만 남기고 압축 |
| 시점 | 새로운 작업을 시작할 때 | 같은 작업을 이어갈 때 |
| 비유 | 노트를 지우고 새로 시작 | 핵심만 남기고 나머지를 지워서 공간 확보 |

### /clear - 새 노트로 시작

하나의 작업을 끝내고 완전히 다른 작업을 시작할 때 사용합니다.

```
이 프로젝트의 README.md를 한국어로 번역해줘
```

번역이 완료된 후, 전혀 다른 작업으로 넘어갑니다.

```
/clear

이 보고서의 3월 매출 데이터를 분석해줘
```

`/clear` 후에는 이전 대화를 모릅니다. "아까 그 파일"이라고 하면 무엇을 가리키는지 알 수 없으므로, 구체적으로 다시 지시합니다.

참고로 `/clear`는 같은 세션 안에서 노트만 비우는 것이고, 세션 자체를 종료하는 것은 아닙니다. 완전히 새 세션으로 시작하고 싶으면 `/exit` 후 `claude`를 다시 실행합니다.

Desktop 앱에서 새 컨텍스트로 시작하려면 각 탭의 새로 만들기 버튼을 클릭합니다.

- Chat → New chat
- Cowork → New task
- Code → New session

### /compact - 요약해서 공간 확보

긴 작업 도중 컨텍스트가 많이 쌓였지만 아직 같은 작업을 이어가야 할 때 사용합니다.

```
/compact
```

뒤에 유지할 내용을 지정하면 더 정확하게 압축됩니다.

```
/compact API 변경사항에 집중해줘

/compact 수정한 파일 목록과 남은 작업을 유지해줘
```

압축이 실행되면 대화 내용이 구조화된 요약으로 교체됩니다.

| 디스크에서 다시 읽어 재주입 | 요약에 포함 | 사라짐 |
|---------------------------|------------|--------|
| 시스템 프롬프트 | 사용자 요청과 의도 | 전체 도구 출력 (파일 내용, 명령 결과) |
| CLAUDE.md (글로벌 + 프로젝트) | 검토/수정한 파일과 중요 코드 조각 | 중간 추론 과정 |
| Auto Memory | 에러와 해결 방법 | 스킬 설명 목록 (호출한 스킬만 보존) |
| 환경 정보, MCP 도구 목록 | 대기 중인 작업과 현재 상태 | |

> 대화에서만 한 지시는 압축 시 사라질 수 있습니다. "한국어로 응답해줘"같은 규칙은 CLAUDE.md에 적어두면 압축 후에도 유실되지 않습니다.

## 컨텍스트 용량 확인 (/context)

현재 컨텍스트가 얼마나 차 있는지 확인하려면 `/context`를 입력합니다.

```
/context
```

맨 위 줄에서 전체 용량과 현재 사용량을 확인합니다. **70%를 넘으면 `/compact`를 고려합니다.**

컨텍스트에는 대화 내용만 들어가는 것이 아닙니다.

| 항목 | 들어가는 내용 | 로드 시점 |
|------|--------------|-----------|
| System prompt | Claude의 기본 동작 지시 | 세션 시작 |
| System tools | 도구 정의 (파일 읽기, 편집, 검색 등) | 세션 시작 |
| Memory files | CLAUDE.md, Auto Memory | 세션 시작 |
| Skills | 스킬 이름과 설명 (전체 내용은 사용 시 로드) | 세션 시작 |
| MCP servers | 연결된 MCP 서버의 도구 정의와 스키마 | 세션 시작 |
| Messages | 대화 내용, 파일 내용, 명령 실행 결과 | 대화 중 |

CLAUDE.md가 길거나 MCP 서버가 많으면 기본 사용량이 늘어나므로, `/mcp`로 서버별 토큰 비용을 확인하고 안 쓰는 서버는 연결을 끊는 것이 좋습니다.

## 자동 압축과 비용

컨텍스트의 약 95%가 차면 자동 압축이 실행됩니다. 95%까지 쌓인 컨텍스트를 요약하는 것보다, 50%에서 미리 압축하는 것이 비용이 적습니다. 수동으로 `/compact`를 사용하면 유지할 내용을 직접 지정할 수 있어 자동 압축보다 더 정확합니다.

같은 문제에 대해 Claude를 두 번 이상 수정했다면, 실패한 시도들이 컨텍스트를 어지럽히고 있는 것입니다. `/clear` 후 배운 것을 반영한 새 프롬프트로 다시 시작하는 것이 더 효과적입니다.

## 압축 시점 조절

더 일찍 압축하고 싶다면 환경변수로 기준을 낮출 수 있습니다.

```bash
# 50%에서 자동 압축 실행 (기본값: 95%)
export CLAUDE_AUTOCOMPACT_PCT_OVERRIDE=50

# 1M 모델에서 500K 기준으로 압축 (나머지 500K 구간의 비용 절감)
export CLAUDE_CODE_AUTO_COMPACT_WINDOW=500000
```

## 압축 시 유지할 내용 지정

CLAUDE.md에 압축 지시를 추가하면 자동 압축에서도 해당 내용이 보존됩니다.

```markdown
# Compact Instructions

압축 시 수정된 파일 목록과 테스트 명령어를 항상 유지할 것
```

`/compact`나 `/clear` 후에도 CLAUDE.md의 내용은 항상 디스크에서 다시 읽혀 컨텍스트에 주입됩니다. 매번 반복해야 하는 중요한 규칙은 대화가 아닌 CLAUDE.md에 적어두면 압축 후에도 유실되지 않습니다.
