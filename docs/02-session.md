# 02. 대화와 세션

설치를 마쳤으니 매일 쓰는 기본기를 익힙니다.

## 대화 시작하기

CLI에서 `claude`를 실행하면 새 세션이 시작됩니다. 초기 프롬프트를 함께 넘길 수도 있습니다.

```bash
# 대화형 세션 시작
claude

# 프롬프트와 함께 시작
claude "이 프로젝트 구조를 설명해줘"
```

Desktop 앱에서는 Code 탭을 선택하고 프로젝트 폴더를 지정한 뒤 입력하면 됩니다.

## 세션 이어하기

세션을 종료한 뒤에도 이전 대화를 이어서 할 수 있습니다.

### 1. 마지막 대화 이어하기

`/exit`으로 세션을 종료한 뒤, `-c` 플래그로 다시 시작합니다.

```bash
claude -c
```

직전 대화가 그대로 이어집니다.

### 2. 현재 세션에 이름 붙이기

대화 중에 `/rename`으로 이름을 붙여둡니다.

```
/rename 버그수정
```

### 3. 이름으로 세션 재개하기

종료 후, 이름으로 바로 재개할 수 있습니다.

```bash
claude -r "버그수정"
```

이전 세션 목록을 보고 싶으면 대화 중에 `/resume`을 입력합니다.

## 파일 참조 (@)

대화 중에 특정 파일을 언급하려면 `@`를 입력합니다.

```
@src/main.js 이 파일의 구조를 설명해줘
```

`@`를 입력하면 자동완성으로 파일 목록이 나타납니다. 경로를 일일이 외울 필요 없이 선택하면 됩니다.

## 슬래시 명령어

`/`를 입력하면 사용 가능한 명령어 목록이 나타납니다. 자주 쓰는 것만 정리합니다.

| 명령어 | 역할 |
|--------|------|
| `/help` | 도움말 |
| `/exit` | 세션 종료 |
| `/model` | 모델 변경 (Opus, Sonnet, Haiku) |
| `/voice` | 음성 입력 모드 (스페이스바 길게 눌러 녹음) |
| `/resume` | 이전 세션 목록에서 선택하여 재개 |
| `/rename` | 현재 세션에 이름 붙이기 |
| `/clear` | 대화 이력 초기화 |
| `/compact` | 대화를 핵심만 남기고 압축 |

지금 다 외울 필요는 없습니다. 사용하다 보면 자연스럽게 익숙해집니다. `/`를 입력하면 언제든 전체 목록을 확인할 수 있습니다.

## 여러 줄 입력

긴 지시를 할 때 줄바꿈이 필요합니다. `Enter`를 누르면 전송되므로, 줄바꿈에는 별도 방법을 씁니다.

| 방법 | 단축키 |
|------|--------|
| 모든 터미널 | `\` + Enter |
| macOS 기본 | Option+Enter |
| iTerm2, WezTerm, Ghostty, Kitty | Shift+Enter |

그 외 터미널에서 Shift+Enter를 쓰려면 `/terminal-setup`을 실행하여 설정합니다.

## 되돌리기 (체크포인트)

Claude가 파일을 수정할 때마다 자동으로 체크포인트를 만듭니다. 게임의 세이브 포인트와 같습니다. 결과가 마음에 들지 않으면 `Esc`를 두 번 누릅니다.

```
Esc Esc
```

되감기 메뉴가 나타나면 화살표 키로 되돌릴 시점을 선택합니다. 선택 후 복원 옵션이 표시됩니다.

- **Restore code and conversation** - 코드와 대화를 모두 해당 시점으로 되돌림
- **Restore code** - 파일만 되돌리고 대화 맥락은 유지 (가장 자주 사용)
- **Restore conversation** - 대화만 되돌리고 파일은 그대로
- **Summarize from here** - 해당 시점부터 대화를 요약으로 압축
- **Never mind** - 취소

`/rewind`를 입력해도 같은 메뉴가 나타납니다. 되돌리기 횟수에 제한은 없으니, 자유롭게 실험하고 마음에 들지 않으면 되돌리면 됩니다.

## 세션 자동 삭제 설정

세션 기록은 나중에 이어하거나 참고할 수 있는 중요한 자산입니다. 기본값은 30일 후 자동 삭제이므로, 1000년(365 x 1000 = 365000일)으로 설정하는 것을 권장합니다.

Claude에게 바로 부탁하면 됩니다.

```
세션 보관 기간을 365000일로 설정해줘. ~/.claude/settings.json의 cleanupPeriodDays 값을 변경해줘.
```

직접 수정하려면 `~/.claude/settings.json` 파일을 열고 아래 값을 추가합니다.

```json
{
  "cleanupPeriodDays": 365000
}
```

`0`으로 설정하면 시작할 때마다 모든 기록을 삭제하고, 세션 저장 자체를 비활성화합니다.

> **주의사항**
> - 체크포인트는 세션 종료 후에도 30일간 유지됩니다. 장기적인 버전 관리는 Git 커밋을 사용합니다
> - 되돌리기는 Claude가 수정한 파일에만 적용됩니다. 사용자가 직접 수정한 내용은 별도로 관리합니다
