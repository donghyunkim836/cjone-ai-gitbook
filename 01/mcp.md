# MCP

## MCP 서버(외부 서비스 연결) 활용 가이드

> Claude가 Notion, Gmail, Slack 같은 외부 서비스에 직접 접속해서 일할 수 있게 해주는 기능

***

### MCP란?

Claude는 기본 상태에서는 대화창에 붙여넣은 내용만 다룰 수 있습니다. 하지만 **MCP(외부 서비스 연결)** 를 설정하면, Claude가 실제 서비스에 접속해서 데이터를 가져오거나 작업을 수행할 수 있습니다.

> **비유하자면** — 새로 온 직원에게 회사 Notion 계정과 Gmail 접근권을 부여하는 것과 같습니다. 접근권이 생기면 직접 들어가서 내용을 읽고, 정리하고, 작성할 수 있게 됩니다.

**연결하면 이런 일이 가능해집니다:**

| 연결 서비스            | 할 수 있는 것                        |
| ----------------- | ------------------------------- |
| **Notion**        | 페이지 읽기, 회의록 자동 저장, 데이터베이스 조회    |
| **Gmail**         | 받은 메일 요약, 답장 초안 작성, 발송 대기 목록 정리 |
| **Google Drive**  | 파일 검색, 내용 요약, 문서 생성             |
| **Slack**         | 채널 내용 요약, 메시지 초안 작성             |
| **Jira / Linear** | 이슈 조회, 상태 업데이트, 티켓 생성           |
| **사내 데이터베이스**     | 고객 정보 조회, 실적 데이터 분석             |

***

### 연결 방법 (데스크탑 앱 / 웹)

#### 데스크탑 앱(Cowork) 기준

1. 앱 상단 또는 사이드바에서 **설정(Settings)** 을 엽니다
2. **Connectors** 또는 **연결된 서비스** 메뉴를 선택합니다
3. 연결하고 싶은 서비스를 목록에서 찾아 클릭합니다
4. 해당 서비스의 로그인 페이지가 열리면 계정으로 로그인합니다
5. 권한 동의 후 연결이 완료됩니다

#### 웹(claude.ai) 기준

1. 오른쪽 상단 프로필 → **Settings** 로 진입합니다
2. **Integrations** 탭을 선택합니다
3. 원하는 서비스를 찾아 **Connect** 버튼을 클릭합니다
4. 브라우저에서 해당 서비스로 이동해 로그인 및 권한 동의를 완료합니다

> 연결은 **한 번만** 하면 됩니다. 이후 Claude와의 모든 대화에서 자동으로 사용됩니다.

***

### 연결 후 사용 방법

별도의 명령어 없이 평소처럼 대화하면 됩니다. Claude가 필요한 서비스에 자동으로 접근해 처리합니다.

#### 예시 — Notion 연결 후

```
오늘 오후 팀 회의 내용을 Notion에 회의록 페이지로 저장해줘.
내용: 신제품 론칭 일정 확정 (8월 15일), 마케팅팀에 소재 요청,
다음 회의는 6월 10일
```

```
Notion에서 '2025 하반기 전략' 페이지 찾아서 주요 내용 요약해줘
```

#### 예시 — Gmail 연결 후

```
오늘 받은 미읽음 메일 중 중요한 것들 요약해줘
```

```
김부장님께 다음 내용으로 메일 초안 만들어줘.
내용: 다음 주 화요일 오전 10시 회의 일정 확인 요청
```

#### 예시 — Google Drive 연결 후

```
드라이브에서 '영업 현황' 관련 파일 찾아서 최신 수치 알려줘
```

```
지난달에 작성한 제안서 파일을 요약해줘
```

### 실전 활용 시나리오

#### 시나리오 1 — 하루 시작 브리핑 자동화

아침마다 이렇게 요청하면:

```
오늘 일정 정리해줘.
- Gmail에서 어제 저녁부터 온 중요 메일
- Notion 팀 캘린더에서 오늘 회의 일정
- Slack에서 내가 멘션된 내용
한 화면에 정리해서 보여줘
```

여러 서비스를 한 번에 조회해서 오늘의 우선순위를 정리해줍니다.

#### 시나리오 2 — 회의 직후 자동 정리

```
방금 끝난 회의 내용이야. Notion 팀 회의록 데이터베이스에 저장해줘.
안건, 결정사항, 액션아이템(담당자/기한) 형식으로.

[회의 내용 붙여넣기]
```

#### 시나리오 3 — 이메일 처리 효율화

```
오늘 받은 메일 중 답장이 필요한 것들 목록 뽑아줘.
우선순위 순으로 정리하고, 각각 어떤 내용으로 답장하면 좋을지 한 줄씩 제안해줘.
```

***

### 연결 범위와 권한

MCP 연결을 통해 Claude가 서비스에 접근할 때는 **본인이 해당 서비스에서 갖고 있는 권한과 동일한 범위**에서만 동작합니다.

* 본인이 볼 수 없는 문서는 Claude도 볼 수 없습니다
* 본인이 수정할 수 없는 항목은 Claude도 수정할 수 없습니다
* 연결 해제는 언제든지 설정에서 가능합니다

***

### 팀 단위 연결 (관리자 설정)

Team/Enterprise 플랜에서는 관리자가 팀 전체에 MCP 연결을 일괄 적용할 수 있습니다. 팀원이 개별적으로 연결하지 않아도, 관리자가 설정한 서비스는 모든 팀원의 Claude에서 자동으로 사용 가능해집니다.

사내 MCP 연결 설정이 필요하다면 **AI 퍼실리테이터 또는 IT 담당자**에게 문의하세요.

***

### 알아두면 좋은 것

* MCP 연결은 **실제 서비스에 접근**합니다. 민감한 정보가 있는 서비스는 연결 전에 꼭 확인하세요.
* 연결된 서비스 목록은 설정 → Connectors에서 언제든 확인하고 해제할 수 있습니다.
* 서비스 연결 후 Claude가 해당 서비스에 접근할 때 사용 내역이 남습니다.
* 업무와 무관한 개인 서비스 연결은 지양하고, 필요한 것만 연결해두는 것을 권장합니다.
